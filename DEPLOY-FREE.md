# Host MENSWEAR 100% Free

Deploy your Bagisto store at **$0/month** using:

| Service | Provider | Cost | Why |
|---------|----------|------|-----|
| Web app | [Render](https://render.com) | Free | Docker hosting, custom domain |
| Database | [Neon](https://neon.tech) | Free | PostgreSQL that **does not expire** (unlike Render's free DB) |

> **Tip:** Open the site 1–2 minutes before your client demo. Render free tier sleeps after 15 minutes of inactivity and takes ~60 seconds to wake up.

## Step 1 — Push code to GitHub

```bash
cd /path/to/shuup-project
git init
git add .
git commit -m "MensWear store with free hosting config"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/menswear-store.git
git push -u origin main
```

---

## Step 2 — Create free PostgreSQL on Neon

1. Sign up at [neon.tech](https://neon.tech) (no credit card required).
2. Click **New Project** → name it `menswear`.
3. Copy the **connection string** (starts with `postgresql://...`).
4. Keep this tab open — you'll paste it into Render next.

Neon's free tier includes ~0.5 GB storage and does **not** delete your database after 30 days.

---

## Step 3 — Deploy on Render

1. Sign up at [render.com](https://render.com) (no credit card for free tier).
2. Click **New +** → **Blueprint**.
3. Connect your GitHub account and select the `menswear-store` repo.
4. Render detects `render.yaml` — click **Apply**.

### Set required environment variables

In the Render dashboard → your service → **Environment**:

| Key | Value |
|-----|-------|
| `DATABASE_URL` | Your Neon connection string from Step 2 |
| `APP_URL` | `https://menswear-store.onrender.com` (use your actual Render URL) |
| `RUN_SEED` | `true` (first deploy only) |

`APP_KEY` is auto-generated by Render if you used the Blueprint.

5. Click **Manual Deploy** → **Deploy latest commit**.
6. Wait 10–15 minutes for the first build (Bagisto is large).

### After first successful deploy

1. Visit `https://YOUR-APP.onrender.com` — storefront should load.
2. Visit `https://YOUR-APP.onrender.com/admin` — admin panel.
3. **Change `RUN_SEED` to `false`** in Environment (prevents re-seeding on restart).
4. Click **Save** and redeploy.

## Troubleshooting

### Build fails on Render
- Check **Logs** tab for the error.
- Ensure `DATABASE_URL` is set before deploy completes.

### 500 error after deploy
- Verify `APP_URL` matches your Render URL exactly (include `https://`).
- Check logs: Render Dashboard → Logs.

### Database connection failed
- Neon connection string must include `?sslmode=require`.
- Use the **pooled** connection string from Neon if the direct one times out.

### Site is slow / blank page
- Free tier is waking up — wait 60 seconds and refresh.

## Optional: keep the site awake (still free)

Use a free cron ping service (e.g. [UptimeRobot](https://uptimerobot.com)) to hit your URL every 14 minutes. This prevents sleep during business hours.

## Local Docker test (before deploying)

```bash
docker build -t menswear .
docker run -p 8080:10000 \
  -e PORT=10000 \
  -e DATABASE_URL="your-neon-url" \
  -e APP_KEY="base64:YOUR_KEY" \
  -e APP_URL="http://localhost:8080" \
  -e RUN_SEED=true \
  menswear
```

Open http://localhost:8080
